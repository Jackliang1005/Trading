# 飞书交易指令说明

## 结构

现在改成纯事件驱动：

1. [飞书交易Webhook.py](/root/.openclaw/workspace/trading/飞书交易Webhook.py)
   是兼容入口，实际代理到 skill。

2. [SKILL.md](/root/.openclaw/workspace/skills/feishu-trading-webhook/SKILL.md)
   定义 skill 的用途和接入方式。

3. [feishu_trading_webhook.py](/root/.openclaw/workspace/skills/feishu-trading-webhook/scripts/feishu_trading_webhook.py)
   直接接收飞书事件、直接更新计划、直接飞书回执。

4. [trading_command_core.py](/root/.openclaw/workspace/skills/feishu-trading-webhook/scripts/trading_command_core.py)
   负责解析指令和修改 [今日交易计划.json](/root/.openclaw/workspace/trading/今日交易计划.json)。

## 当前主流程

当前推荐按这条链路使用：

1. 维护初始持仓库
2. 查看观察池和今日计划
3. 查看仓位状态和指令簿
4. 需要时手动下达/直接执行

旧的 `T 开启 / T 关闭 / T 设置` 仍然兼容，但不再是推荐主路径。

### 帮助
```text
T 命令
```

或：

```text
T 帮助
```

### 查看今日状态
```text
T 状态
```

### 查看初始持仓数据库
```text
T 初始持仓
```

说明：
- 显示账户概览、底仓、可用余额、成本价、策略
- 这份库是系统初始化当日仓位和预算分配的起点

### 更新初始持仓数据库
```text
T 更新持仓 300475 200 158.6426 200 143.96 158.643
```

格式：
```text
T 更新持仓 <证券代码> <底仓> <成本价> [可用余额] [现价] [保本价]
```

说明：
- 优先用于修正系统的真实底仓基线
- 后续可用 `T 重建持仓` 把它重建到当天仓位状态

### 按初始持仓库重建当日仓位
```text
T 重建持仓
```

说明：
- 会重建当天 `portfolio_state`
- 适合你修正初始持仓库后重新同步系统

### 查看观察池
```text
T 观察池
```

说明：
- 显示 `focus / watch / avoid`
- 同时显示 `AM/PM 模式`
- 同时显示 `买预算 / 卖预算 / 建议股数`
- 同时显示事件标签和新闻解释

### 查看今日做T计划
```text
T 今日计划
```

说明：
- 显示 selector 回写后的当日计划
- 重点看 `selection_action`、`selection_am_mode`、`selection_pm_mode`

### 查看今天的仓位状态
```text
T 仓位
```

说明：
- 会显示每只票的底仓、今买、今卖、当前仓位
- 重点看 `可卖` 和 `可回补`

### 查看今天的指令簿
```text
T 指令簿
```

### 查看今天的执行回执汇总
```text
T 成交日报
```

### 查看今天的决策统计
```text
T 决策日报
```

### 查看模式复盘
```text
T 模式复盘
```

说明：
- 查看不同 `AM/PM mode` 当天触发了多少买卖和自动候选

## 飞书侧需要配置什么

在飞书开放平台里，把机器人的事件订阅回调地址指向：

```text
http://你的主机地址:8788/feishu/trading
```

至少订阅文本消息相关事件。

如果飞书要求 challenge 校验，这个 webhook 已经支持。

---

## 本地怎么启动 webhook

```bash
cd /root/.openclaw/workspace/trading
python3 飞书交易Webhook.py --port 8788
```

如果你要直接从 skill 路径运行：

```bash
cd /root/.openclaw/workspace/skills/feishu-trading-webhook
TRADING_BASE_DIR=/root/.openclaw/workspace/trading python3 scripts/feishu_trading_webhook.py --port 8788
```

建议后面做成 `systemd` 服务。

## 当前边界

- OpenClaw 当前可以发飞书消息，但不能直接 `read` 飞书消息，所以这里用 skill webhook 入站。
- 飞书消息一到就立即更新当天计划，不再依赖轮询。
- 现在已经支持通过 `qmt2http` 直接下单，但默认不开自动盘中下单。
- 当前 `qmt2http` 稳定能力以 `buy / sell / cancel` 为主，查询类接口按尽力支持处理。
- 若要启用盘中信号直连执行，请在 [做T监控配置.json](/root/.openclaw/workspace/trading/做T监控配置.json) 里设置 `qmt2http.auto_trade_enabled=true`。
- 如果 webhook 没启动或飞书回调没配好，监控还能跑，但你无法从飞书实时改计划。
